# Assets

Use this directory for controlled asset intake and metadata manifests.

Structure:

- `intake/brand/`
  raw Daysi logo and brand files
- `intake/marketing/`
  raw marketing imagery
- `intake/before-after/`
  raw before-and-after imagery
- `manifests/`
  CSV metadata templates and completed manifests

Rules:

- do not put patient names or contact details in filenames
- do not treat git as the long-term storage location for raw media libraries
- before-and-after assets need consent metadata before public use

See `docs/asset-onboarding-spec.md` for the intake workflow.
